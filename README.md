# Portfolio Site

Minimal portfolio site generated from a compact native codebase.

## Design Targets

- predictable output
- compact native implementation
- no runtime dependencies
- restrained visual surface

## Repo Layout

```text
.
├── .github/workflows/ci.yml
├── Makefile
├── src/
│   ├── main.c      # socket server + export mode
│   ├── page.c      # HTML renderer + project data
│   └── page.h
└── scripts/
    └── smoke.sh    # basic endpoint checks
```

## Quick Start

```bash
make
make run
```

Open `http://127.0.0.1:8080`.

## Export Static HTML (GitHub Pages-friendly)

GitHub Pages cannot run native binaries, so generate static output straight into `docs/`:

```bash
make export
```

This writes `docs/index.html`, generated by the same renderer used for the local server.
The `docs/` folder is the deployable site.

## Customize Content

Edit `PROJECTS` in `src/page.c`:

- `name`
- `stack`
- `summary`
- `repo`

Also update the hero copy in `render_portfolio_html`:

```c
<h1>Quiet surfaces. Exact internals.</h1>
```

## Validation

```bash
make smoke
```

## Benchmarking

```bash
make bench
```

Measured response notes live in `LATENCY.md`.

## Suggested GitHub Setup

1. Push this repo.
2. In GitHub, open `Settings -> Pages`.
3. Under `Build and deployment`, set the source to `Deploy from a branch`.
4. Choose branch `main` and folder `/docs`.
5. Later, point your custom domain at your dedicated server when you are ready to move off Pages.
