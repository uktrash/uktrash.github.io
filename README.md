# Portfolio Site

Minimal portfolio site generated from a compact native codebase.

## Design Targets

- predictable output
- compact native implementation
- no runtime dependencies
- restrained visual surface

## Repo Layout

```text
.
├── .github/workflows/ci.yml
├── Makefile
├── src/
│   ├── main.c      # socket server + export mode
│   ├── page.c      # HTML renderer + project data
│   └── page.h
└── scripts/
    └── smoke.sh    # basic endpoint checks
```

## Quick Start

```bash
make
make run
```

Open `http://127.0.0.1:8080`.

## Export Static HTML (GitHub Pages-friendly)

GitHub Pages cannot run native binaries, so generate static output straight into `docs/`:

```bash
make export
```

This writes both `docs/index.html` and a mirrored root `index.html`, generated by the same renderer used for the local server.
That means GitHub Pages will serve the portfolio whether the repo is configured to deploy from `/docs` or from the branch root.

## Customize Content

Edit `PROJECTS` in `src/page.c`:

- `name`
- `stack`
- `summary`
- `repo`

Also update the hero copy in `render_portfolio_html`:

```c
<h1>Quiet surfaces. Exact internals.</h1>
```

## Suggested GitHub Setup

### One-Click Deploy

1. Push this repo.
2. In GitHub, open `Settings -> Pages`.
3. Under `Build and deployment`, set the source to `GitHub Actions`.
4. Push to `main`, or run the `pages` workflow manually from the `Actions` tab.

### Manual Fallback

1. Run `make export` so both `docs/index.html` and `index.html` are current.
2. Push the generated files.
3. Under `Settings -> Pages`, choose `Deploy from a branch`.
4. Select branch `main`.
5. Choose either:
   - `/docs` if you want the generated site served from `docs/`
   - `/ (root)` if you want GitHub to serve the mirrored root `index.html`

If Pages is pointed at the branch root without a root `index.html`, GitHub may render `README.md` instead, which is the mismatch you were seeing.

Later, point your custom domain at your dedicated server when you are ready to move off Pages.
